# Архитектура и работа Birthday Bot (Proxy Edition)

Этот документ описывает, как организована отправка уведомлений в Telegram в условиях сетевых блокировок.

## Общая схема
`[Бот (C)]` -> `[Локальный порт 1080 (SOCKS5)]` -> `[SSH-туннель]` -> `[vpn-server (Нидерланды)]` -> `[Telegram API]`

---

## 1. Компоненты системы

### А. Бинарный файл `birthday_bot` (C)
Это основной исполнитель. Он подключается к PostgreSQL, находит именинников и формирует сообщения.
*   **Особенность**: В коде используется библиотека `libcurl` с настроенной опцией `CURLOPT_PROXY`.
*   **Настройка**: Запросы отправляются на `socks5h://127.0.0.1:1080`. Префикс `socks5h` важен: он заставляет прокси-сервер выполнять DNS-разрешение (`api.telegram.org`) на своей стороне (в Европе), обходя локальные блокировки DNS.

### Б. SSH-туннель (Прокси)
Создается командой:
`ssh -f -N -D 127.0.0.1:1080 vpn-server`
*   `-D 127.0.0.1:1080`: Динамическая переадресация портов. Создает локальный SOCKS-сервер.
*   `-f`: Запуск в фоновом режиме.
*   `-N`: Не выполнять удаленные команды (только туннелирование).
*   **vpn-server**: Удаленный сервер с европейским IP, имеющий доступ к Telegram.

### В. Скрипт-обертка `run_bot.sh` (Bash)
Обеспечивает отказоустойчивость (Self-healing).
1.  **Проверка**: Перед запуском бота делает тестовый запрос `curl` через прокси к Telegram.
2.  **Восстановление**: Если Telegram недоступен, скрипт убивает старый SSH-процесс и поднимает туннель заново.
3.  **Запуск**: После подтверждения связи запускает `birthday_bot`.

---

## 2. Мониторинг и управление

### Проверка прокси
Выполните команду для проверки открытого порта:
```bash
ss -nltp | grep 1080
```

### Тестирование связи вручную
Проверка, проходит ли трафик до Telegram через прокси:
```bash
curl -x socks5h://127.0.0.1:1080 https://api.telegram.org/bot<TOKEN>/getMe
```

### Логирование
Все события (проверка прокси, перезапуск туннеля, ошибки бота) записываются в файл:
`~/birthday_bot/birthday_bot.log`

---

## 3. Расписание (Cron)
Бот запускается ежедневно в 09:00 через `crontab` пользователя root.
Задание выглядит так:
`0 9 * * * cd /root/birthday_bot && ./run_bot.sh >> birthday_bot.log 2>&1`
